# Part 0 Foundations — side topics every engineer needs

These five chapters cover concepts that the rest of the book *assumes* you know. If you've programmed before, you can skip them. If chapters in Part 0 left you unsure what "10 in hex" or "the AND of two bits" or "what `<<` does" means, **read these now.**

Read them in any order. Each is short.

| # | Chapter | What it teaches |
|---|---|---|
| A1 | [Numbers — binary, hex, decimal](A1-numbers.md) | Why hardware uses base 2 / base 16, and how to read both |
| A2 | [Bits, bytes, words](A2-bits-and-bytes.md) | What "8-bit" / "32-bit" really mean, KB vs KiB |
| A3 | [Boolean logic](A3-boolean-logic.md) | AND / OR / NOT / XOR — the alphabet of digital hardware |
| A4 | [Shell power tour](A4-shell-power-tour.md) | Pipes, redirects, grep, env vars, scripting basics |
| A5 | [Character encoding](A5-character-encoding.md) | ASCII, UTF-8, why text is harder than it looks |

After these, return to whatever chapter you came from.
